Exercise I.8

Play with color, noiseDetail(), and the rate at which xoff and yoff are incremented to achieve different visual effects.

Solution

Wow, OK, this was not easy. The canvas is much smaller because at the 800 by 800 pixel size my PC was having issues. This exampleis working and you can change the values as you like to see the differences.

First off, the code to set the color for the pixels in p5.js is significantly different than in normal processing. I used the code directly from the reference document and I don't fully understand it. The code is for an individual pixel, so I put it in the function setPixelColor() which accepts the x, y, and both offsets as an argument. 

setPixelColor(x, y, xoff, yoff){
  let d = pixelDensity();

  for(let i = 0; i < d; i++) {
    for (let j = 0; j < d; j++) {
      let index = 4 * ((y * d + j) * width * d + (x * d + i));

      let bright = map(noise(xoff, yoff), 0, 1, 0, 255);
      let c = color(bright);

      pixels[index] = c._getRed();
      pixels[index + 1] = c._getGreen();
      pixels[index + 2] = c._getBlue();
      pixels[index + 3] = c._getAlpha();
    }
  }
}

I also had to use the debugger to get the methods to read the red, green, blue, and alpha values from the color. They are private but this still worked. In the update() function I loop through the pixels like in the book and make a call to the function. Note the x and y offsets are passed into the function and are updated outside of the setPixelColor() function. 

update() {
  loadPixels();

  let xoff = 0.0;

  for (let x = 0; x < width; x++) {
    let yoff = 0.0;

    for (let y = 0; y < height; y++) {
      this.setPixelColor(x, y, xoff, yoff);

      yoff += 0.1;
    }

    xoff += 0.1
  }

  updatePixels();
}
